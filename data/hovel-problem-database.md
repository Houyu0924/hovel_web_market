# HOVEL 悩みデータベース v1.0

最終更新日: 2026-07-22

## 目的

HOVELが解決すべき顧客課題を継続的に蓄積し、記事、動画、HOVEL Lab、AI、商品、サービスの企画へ接続するための基礎データベースです。

HOVELは、お客様にとって最適な解決策を誠実に提供し続けます。すべてを自社だけで解決しようとはせず、必要に応じて専門家、医療機関、公的窓口、他社商品・サービスも選択肢に含めます。

## 運用原則

1. 悩みを見つけても、すぐに商品や解決策へ飛ばない。
2. 誰が、いつ、どの場面で、何に困っているかを記録する。
3. 表面的な症状と根本原因を分ける。
4. 解決策は一つに固定せず、複数のアプローチを比較する。
5. 医療・法律・金融など専門資格が必要な領域では対応範囲を明示する。
6. 公開情報だけで断定できない効果は、仮説または検証候補として扱う。
7. 実生活で発見した悩みを随時追加し、重複は統合する。

## ステータス

- `未整理`: 悩みを登録した段階
- `調査待ち`: 市場・原因・既存解決策を調べる
- `検証候補`: HOVEL Labで検証する価値がある
- `企画中`: 記事・動画・商品などを設計中
- `公開済み`: 解決コンテンツを公開済み
- `継続改善`: 公開後の反応や新しい知見を反映中
- `専門家案内`: HOVEL単独では扱わず専門家へ接続する

## 優先度評価軸

各軸を1〜5点で評価します。総合点だけで機械的に決めず、目的別に見ます。

| 評価軸 | 見る内容 |
|---|---|
| 深刻度 | 生活・健康・仕事への悪影響 |
| 緊急度 | 放置した場合のリスクや期限 |
| 対象者数 | 同じ悩みを持つ人の多さ |
| 発生頻度 | 日常で繰り返し発生するか |
| 人生影響度 | 改善が人生全体へ波及するか |
| HOVEL適合度 | 25〜35歳男性会社員との親和性 |
| 解決可能性 | 根拠を持って有用な選択肢を出せるか |
| 独自性 | 実験・比較・AI活用で差別化できるか |
| 維持支援性 | 一度の改善でなく継続支援できるか |
| 事業性 | 継続可能な価値提供と収益につながるか |

## 解決アプローチ

悩みごとに複数の方法を組み合わせます。

- `知識`: 原因、選び方、誤解の修正
- `診断`: チェックリスト、記録、AI診断
- `環境`: 道具、配置、服装、生活環境の変更
- `行動`: 手順、ルーティン、習慣設計
- `商品`: 比較、選定、使用方法
- `AI`: 計画、記録、振り返り、自動化
- `伴走`: 定期レビュー、コミュニティ、コーチング
- `専門家`: 医療、法律、金融、心理などへの案内
- `撤退`: 不要な支出や効果の薄い行動をやめる
